API Info

Introduction

My API allows direct interactions with the server-side part of the site without the need for my web pages. This makes it easy for developers to build their own front-end to the site.

The API follows the REST approach. Resources are determined by the URL and the action you want to take is determined by the HTTP request type. URLs should intuitively describe the resource; If you make a GET request to http://{site}/api/comments/today/ you can expect to receive comments from today. For GET requests you will also receive information about related resources enabling you to navigate the API. All data sent between client and server is JSON.

Below is a guide on how to use my API with examples.

How to use the API

By sending a GET request to the API's root you can get links to the available resources:

Request:

GET: http://{website}/api/

Response:

{
    "links": [
        {
            "url": "/api/comments/",
            "type": "GET"
        }
    ]
}

At the moment comments are the only resource available, as seen in the JSON response.

Comments

The request below will return further information about the comments as well as a link back to the api root. To access comments you also need to provide a filter like "today" or "3-days-ago". This information is included in the response:

Request:

GET: http://{website}/api/comments/

Response:

{
    "links": [
        {
            "url": "/api/",
            "type": "GET"
        },
        {
            "url": "/api/comments/",
	    "type": "POST"
        },
        {
            "url": "/api/comments/past-week/",
	    "type": "GET"
        },
        {
            "url": "/api/comments/today/",
            "type": "GET"
        },
        {
            "url": "/api/comments/1-day-ago/",
	    "type": "GET"
        },
        {
            "url": "/api/comments/2-day-ago/",
            "type": "GET"
        },
        {
            "url": "/api/comments/3-day-ago/",
            "type": "GET"
        },
        {
            "url": "/api/comments/4-day-ago/",
            "type": "GET"
        },
        {
            "url": "/api/comments/5-day-ago/",
            "type": "GET"
        },
        {
            "url": "/api/comments/6-day-ago/",
            "type": "GET"
        }
    ]
}

Here are two examples of requests which include the filter, and the response they would get.

Request:

GET: http://{website}/api/comments/past-week/
GET: https://matthewpenney.net/api/comments/2-days-ago/

Response:

{
    "links": [
        {
            "url": "/api/comments/",
            "type": "POST"
        },
        {
            "url": "/api/comments/",
            "type": "GET"
        }
    ],
    "comments": [
        {
            "comment": "some comment",
	    "day_posted": "dd/mm/yyyy",
	    "name": "Jeff"
	},
	{
            "comment": "another comment",
	    "day_posted": "dd/mm/yyyy",
	    "name": "Nigel"
        }
    ]
}

You can also use a POST request to post a comment. You will need to include JSON data in the body of the request containing your comment, as shown below:

Request:

POST: http://{website}/api/comments/

Request body:

{
    "comment": "Your comment"
    "name": "Your name"
}
